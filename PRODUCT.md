# Product

## Register

product

## Users

Developers at all experience levels (junior, mid-level, senior) actively learning the Blazor framework for C#. Primary context: mobile, during commutes or focused evening sessions. Secondary: desktop for deeper reference. The user is in deliberate learning mode — motivated, trying to build durable knowledge of a specific technical stack, not passively scrolling.

## Product Purpose

BlazorPath is a swipeable card-based learning app for mastering the Blazor framework in C#. It structures learning across three tiers (Junior → Mid → Senior), delivers one focused concept per card, injects quiz cards every few cards to reinforce retention, auto-builds a personal glossary from encountered abbreviations, and tracks tier-level progress. Success: a developer completes all three tiers with durable recall of key Blazor concepts — and has a searchable glossary to reference later.

## Brand Personality

Editorial, dense, authoritative. Like a brilliant senior dev explaining to a teammate — direct, no fluff, all signal. Three words: sharp, focused, honest.

## Anti-references

- Duolingo (gamified, cartoon aesthetic, confetti dopamine loops)
- Any flashcard app with bright pastels and bouncy animations
- Material Design defaults (too consumer, too colorful)
- SaaS landing pages with gradient CTAs and hero metrics
- Cream/sand/warm-neutral backgrounds — the saturated AI default of 2025-26
- Notion-style "clean white" minimalism (too light for a developer tool used at night)

## Design Principles

1. Signal over decoration — every visual element earns its place by communicating something; decoration is a tax on attention
2. Dense without crowded — pack real content into each card; do not pad for whitespace theater
3. The interface disappears into learning — chrome is minimal, content leads every screen
4. Dark is deliberate — developers read at night; this is not a consumer app; the dark surface is a choice, not a theme
5. Motion conveys state, not personality — the swipe says "done"; nothing else spins for fun

## Accessibility & Inclusion

WCAG AA minimum contrast throughout. Minimum tap target 44×44px. Reduced motion via prefers-reduced-motion (instant transitions replace spring physics). Inter with system-ui fallback. Rem-based font sizing respects system preferences.
